# Проект IT‑компании «Йети»: предсказание возраста посетителей сайтов

В рамках проекта для IT‑компании «Йети» была разработана модель машинного обучения для определения возрастной категории пользователя (5 классов: от «младше 18» до «56+ лет») на основе логов цифровой активности — без использования паспортных данных. Решение ориентировано на повышение эффективности таргетированной рекламы и снижение доли нерелевантных показов.

### Подготовка и качество данных

На этапе подготовки данных были загружены и проверены 6 датасетов, устранены дубликаты, проведена унификация кодировок категориальных признаков (перевод на латиницу для последующей обработки). Ключевой сложностью стало объединение разнородных источников по `user_id` с разной полнотой покрытия: из‑за несовпадения числа уникальных пользователей в отдельных таблицах при соединении возникали пропуски. Они были корректно обработаны заполнением модой с учётом распределений признаков.

Для перехода от логов сессий к пользовательскому уровню была выполнена агрегация `df_visits`: построены счётчики сессий по времени суток и категориям сайтов, рассчитаны отношения (например, `ratio_evening_to_morning`), добавлены признаки энтропии и бинарные индикаторы. Итоговый датасет `df_users_logs` содержит 5826 записей и 34 признака, где целевой переменной выступает `age_category`.

### Анализ данных и отбор признаков

EDA выявил важные закономерности:

- **Дисбаланс целевой переменной**: наибольшая доля у категории «56+ лет» (30.3 %), наименьшая — у «18–25 лет» (8.8 %). Это потребовало применения стратифицированной кросс‑валидации и учёта дисбаланса в обучении.
- **Мультиколлинеарность среди счётных признаков**: коэффициенты корреляции между признаками типа `total_sessions`, `avg_sessions_per_day`, `count_*` достигали 0.8–0.99. Для стабилизации модели было решено использовать логарифмическое преобразование этих переменных и отбор признаков.
- **Информативность отдельных признаков**: по матрице Phik и взаимной информации наибольшую связь с целевой переменной показали `ratio_evening_to_morning`, счётчики посещений отдельных категорий сайтов и признак `ads_activity`.

Отбор признаков проводился в несколько этапов: логарифмирование счётных переменных, One Hot Encoding категориальных признаков, отбор по взаимной информации (для числовых) и критерию $\chi^2$ (для категориальных), финальный отбор — с помощью RFE на базе LogisticRegression. Все этапы были инкапсулированы в единый пайплайн для исключения утечки данных и воспроизводимости результатов.

### Моделирование и результаты

Были протестированы несколько подходов к мультиклассовой классификации, включая варианты с разными стратегиями разбиения классов (One vs Rest, One vs One) и базовыми моделями (LogisticRegression, SVC). Подбор гиперпараметров выполнялся через `GridSearchCV` с кросс‑валидацией и макро‑усреднёнными метриками, что особенно важно при дисбалансе классов.

Лучшей оказалась модель **One vs Rest SVC с ядром RBF и параметром регуляризации C=1**. Её результаты:

- **F1 macro на тестовой выборке: 0.892** (превышает целевой порог 0.75);
- **Accuracy: 0.913**;
- Стабильные показатели по всем классам, без сильного перекоса в сторону мажоритарных групп.

Сравнение с базовой моделью (`DummyClassifier(strategy='stratified')`) показало многократное превосходство: F1 macro у Dummy составил всего 0.191, что подтверждает, что модель действительно извлекает полезные закономерности из данных, а не просто воспроизводит распределение классов.

Наиболее высокие результаты модель демонстрирует для возрастных групп «41–55 лет» и «56+ лет» (F1 = 0.93 и 0.96 соответственно). Наименьшая точность наблюдается для группы «18–25 лет» (precision и recall = 0.77), что, вероятно, связано с более неоднородным поведением пользователей этой возрастной категории и меньшим объёмом данных (support = 103).

### Практическая ценность и дальнейшие шаги

Модель готова к внедрению в продуктивную среду для сегментации аудитории и повышения релевантности рекламных показов.  Модель передается в виде файла `best_estimator_ovr_svm.joblib` и функции **build_df_users_features_func** сборки подготовки единого датасета "признак-объект" в файле `feature_builder.py`.
Применение модели позволит:

- снизить долю нерелевантной рекламы и оптимизировать расход рекламного бюджета;
- повысить конверсию за счёт более точного попадания в целевую аудиторию;
- исключить показ рекламы для взрослых несовершеннолетним.

Таким образом, поставленная задача решена: построена надёжная модель, которая стабильно и точно определяет возрастные категории пользователей по их цифровой активности, соответствует требованиям заказчика по качеству и готова к практическому использованию в маркетинговых сценариях компании «Йети».
